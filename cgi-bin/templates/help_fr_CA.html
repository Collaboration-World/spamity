[% i18n('search') %]

  • Exécute une recherche parmi les messages rejetés [% IF NOT admin %]pour votre ou vos adresses courriel[% END %]selon différents critères.
  • Pour rechercher les messages d'un domaine précis, spécifiez ce domaine dans le champs [% i18n('email/domain') %], sans l'arobas (@) (exemple: domaine.tld).
  • Selon l'installation, une partie ou l'ensemble des messages rejetés sont placés en quarantaine. L'icône d'une enveloppe () apparaît à gauche des messages retenus en quarantaine. Vous pouvez voir un message en quarantaine en cliquant sur l'icône et éventuellement le ré-injecter dans votre compte si vous jugez le message légitime.
  • Les résultats sont groupés par adresse courriel, chronologiquement.[% IF admin %] Entant qu'administrateur, vous pouvez aussi afficher les résultats par date, en commançant par le plus récent.[% END %]

[% i18n('preferences') %]

  • Configurez la politique de filtrage pour votre ou vos adresses courriel. La politique par default est définie par l'administrateur du système. Des politiques prédéfinies additionnelles peuvent être aussi disponibles. Sélectionnez [% i18n('customized') %] si vous désirez modifier chacun des paramètres.
  • [% i18n('tolerence') %] - Certaines méthodes de filtrage peuvent être désactivées. Lorsqu'une méthode est désactivée, il est possible de spécifier une extension courriel (par exemple, spécifier pourriel comme extension courriel change l'adresse du destinataire nom@domaine.tld par nom+pourriel@domaine.tld lorsque le filtre s'applique au message).
  • [% i18n('spam levels') %] - À chaque message est attribué un niveau de pourriel. Trois (3) valeurs engagent des actions distinctes (étiquetage et/ou destruction). Lorsqu'un message est étiqueté, son sujet peut aussi être précédé d'un préfix.
  • [% i18n('lists') %] - Lorsque vous utilisez un politque prédéfinie ou une politique personnalisée, vous pouvez forcer le rejet ou l'acceptation d'adresses (nom@domaine.tld), de domaines (@domaine.tld), ou de noms (nom).
[% END %]

[% i18n('appendix: filter types') %]

  • rbl - Message rejeté par une liste Realtime Blackhole List
  • rhsbl client - Message rejeté par une liste Right Hand Side Block List
  • header date - La date du message n'est pas valide.
  • header suject - Message rejeté pour cause d'un sujet suspect.
  • header x-mailer - Message rejeté pour cause d'un logiciel d'expédition suspect.
  • header content-disposition - Message rejeté pour cause d'un fichier attaché suspect.
  • header content-type - Message rejeté pour cause d'un fichier attaché suspect.
  • body - Message rejeté pour cause d'un contenu suspect.
  • access username - Message rejeté pour cause d'un nom d'expéditeur suspect.
  • virus - Message rejeté par le logiciel d'anti-virus.
  • spam - Message considéré un pourriel par SpamAssassin.
  • banned - Message rejeté pour cause d'un fichier attaché prohibé.
  • helo - Message rejeté pour cause d'une commande HELO manquante ou inacceptable.
